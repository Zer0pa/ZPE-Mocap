# Doc Registry

Canonical registry of the documentation surfaces in `docs/`.
Paths are relative to `docs/`.

| Document | Role | Authority Scope | Status |
| --- | --- | --- | --- |
| `README.md` | Documentation routing index. | Navigation only; authority metrics live in root `README.md`. | Active |
| `ARCHITECTURE.md` | Runtime map and authority routing reference. | Architecture and boundary narrative; defers metrics to root `README.md`. | Active |
| `LEGAL_BOUNDARIES.md` | Legal boundary summary. | Legal notes only; `LICENSE` remains source of truth. | Active |
| `SUPPORT.md` | Support and escalation routes. | Contact and issue routing only. | Active |
| `DOC_REGISTRY.md` | Canonical documentation registry. | Docs inventory and status only. | Active |
