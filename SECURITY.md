# Security

This policy covers vulnerability reporting for ZPE-Mocap. This repo is
currently private staging. `LICENSE` is the legal source of truth; this
file is an operational summary only. Reports are handled with an
evidence-first standard.

## Reporting

| Channel | Use |
| --- | --- |
| `architects@zer0pa.ai` | Private vulnerability reporting |

Include enough evidence for a cold reader to validate the issue:

- A clear description of impact and affected surface.
- Reproduction steps or proof of concept.
- Logs, traces, or artifacts with precise paths or attachments.

**Do not open a public issue for:**

| Category | Examples |
| --- | --- |
| Secrets or credentials | Tokens, keys, passwords |
| Supply-chain or dependency vulnerabilities | Compromised packages, poisoned artifacts |
| Arbitrary code execution or privilege escalation | RCE, sandbox escapes, escalation paths |

If you are unsure whether a report is in scope, send it privately with the
evidence you have and note any gaps.
